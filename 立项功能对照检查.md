# 立项功能对照检查

> 核验基准：`立项书.docx`、`技术可行性与实施方案.md`、`前端与结项展示方案.md`。更新时间：2026-07-13。

## 当前已形成闭环

| 功能 | 状态 | 说明 |
|---|---|---|
| 规则 + NER + 选择性 LLM | 已实现 | 原文 Span、offset 校验、模型失败降级、LLM 重试与并发保护 |
| 八类实体 Schema | 已实现 | PERSON、ORG、LOCATION、ADDRESS、PHONE、EMAIL、ID_CARD、BANK_CARD |
| 三种脱敏策略 | 已实现 | 一致性掩码、确定性伪名、本地泛化 |
| 隐私工作台 | 已实现 | 导入、检测、高亮、复核、改类、增补、边界调整、对照和导出 |
| 人工复核与审计 | 已实现 | 操作会原子更新任务快照，并写入 SQLite 审计记录 |
| 历史与策略 | 已实现 | 任务历史、按实体类型策略持久化；模型运行参数仍由服务器环境变量管理 |
| 评估结果接入 | 已实现 | 无真实实验文件时明确标记演示数据，不再展示硬编码“成果” |
| GPU 部署 | 已实现 | Docker Compose + Nginx + FastAPI + 可选 vLLM/Qwen3-14B-AWQ |

## 部分实现

| 功能 | 当前边界 | 后续建议 |
|---|---|---|
| 多语种 | 首版为中文、英文和中英混合 | 结项材料不要表述为覆盖任意语种 |
| 批处理 | 支持 TXT/CSV/JSON、校验、结果预览和下载，但仍为同步任务 | 增加持久化后台 Job、轮询或 SSE、失败行下载 |
| 评估实验 | 已有 benchmark 入口和前端真实数据契约 | 补 A–F 对照、消融、泄漏率、过度脱敏率、p50/p95 和错误案例 |
| 泛化 | 本地确定性类型泛化 | 后续增加行政区、机构后缀层级；CN-DBpedia 仅作为可选增强 |
| 数据安全 | 历史摘要已脱敏 | SQLite 仍保存任务原文用于偏移复核，生产环境应加保留期限/加密/清除功能 |
| 离线演示 | 规则与轻量识别器可降级运行 | 尚未形成独立“真实结果回放包”管理页面 |

## 尚未实现或明确后置

- LoRA/QLoRA 训练与微调实验；应先依据冻结测试集的错误分析决定是否开展。
- 自然语言策略解析、自定义正则/关键词 CRUD。
- CN-DBpedia 在线实体链接和严格差分隐私证明。
- DOCX/PDF/OCR 版式保持、文件夹批量任务。
- 复杂登录、RBAC、多租户、浏览器插件和桌面客户端。
- 软件著作权和专利申请材料。

## 结项前最高优先级

1. 建立并冻结 gold 测试集，替换所有演示指标。
2. 运行规则、NER、规则+NER、级联+LLM 的公平对照和消融实验。
3. 将同步批处理升级为持久化任务，并记录失败行和真实进度。
4. 增加历史原文保留期限、清除和生产加密策略。
5. 准备在线、轻量降级、离线回放三套答辩预案。

当前版本可定位为“完整度较高、可真实部署的结项原型”，不应宣称原立项书中所有拓展承诺均已完成。
